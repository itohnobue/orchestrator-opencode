# Elixir Pro

## OTP Pattern Selection

| Need | Pattern | Key Constraint |
|------|---------|----------------|
| Stateful process | GenServer | Offload heavy work to Task; handle_info is for messages, handle_continue for post-init work |
| Temporary computation | Task.async | Tasks are linked — caller death kills the task unless using Task.Supervisor |
| Simple key-value state | Agent | Only get/update; no side effects inside Agent functions |
| Independent workers | `one_for_one` Supervisor | Child crash doesn't affect siblings |
| Coupled workers | `one_for_all` Supervisor | All restart when any fails |
| Ordered dependencies | `rest_for_one` Supervisor | Later children restart when earlier ones crash |
| Dynamic processes | DynamicSupervisor + Registry | Use `via_tuple(Registry, ...)` for naming; child_spec `id` collisions silently replace |

## Ecto Query Patterns

| Situation | Approach | Why |
|-----------|----------|-----|
| Avoid N+1 | `Repo.preload` or `Ecto.Query.preload` | `preload` in query binds joins; `Repo.preload` issues separate queries |
| Multi-table atomic writes | `Ecto.Multi` | Composable transactions with rollback — never nest `Repo.transaction` calls |
| Partial updates | `Ecto.Changeset.change/2` | No validation needed for internal updates; skip `cast` if data is trusted |
| Bulk inserts | `Repo.insert_all` | Skips callbacks, changesets, autogenerated timestamps — handle manually |
| Raw SQL | `Ecto.Adapters.SQL.query/4` | Only when Ecto query DSL can't express it; use parameterized queries |

## Non-Obvious Gotchas

- **`with` w/o `else`**: Unmatched clauses silently return the unmatched value — not an error tuple. If `with {:ok, x} <- step1(), do: x` and `step1()` returns `{:error, :enoent}`, the result is `{:error, :enoent}`, not a match error. Always add `else` when callers need to handle failure.
- **`send/2` to dead process**: Succeeds silently — no error, no return. The message is lost. Unlike Erlang's `!` on registered names which raises, Elixir's `send` is always silent.
- **LiveView assigns overhead**: Every key in `socket.assigns` is serialized and sent over WebSocket on every render diff. Large maps, Ecto structs, or data not used in the template inflate every frame. Use `stream/4` for collections, put derived data in `assigns` only when templates consume it.
- **`Enum.each/2` returns `:ok`**: Always. Not the last expression, not a transformed list — `:ok`. Use `Enum.map` for transformations.
- **`unless` with `else`**: Parse order is counterintuitive — `unless a, do: b, else: c` reads as "unless a then b otherwise c" but many readers mistake it. Prefer `if` with negated condition.
- **`defstruct` runtime enforcement**: `%MyStruct{key: val}` checks keys at compile time only for literal maps. `struct(MyStruct, map)` at runtime silently ignores unknown keys and fills missing keys with defaults — no error.
- **`inspect` in production**: `inspect/2` with default opts reveals full struct internals, PIDs, and refs. In log messages or error reports, use `inspect(value, structs: false)` or custom `String.Chars` impl.
- **Dialyzer is optimistic**: Dialyzer assumes success — it reports only type combinations it can prove will fail. A missing Dialyzer warning does NOT mean types are correct. Run Dialyzer in CI and treat warnings as compile errors.
- **`Process.exit(pid, :kill)`**: Cannot be trapped by `Process.flag(:trap_exit, true)`. Only `:kill` has this property — `:shutdown` and other signals are trappable.
- **Module attribute accumulation**: `@attr value` accumulates in a list when used multiple times before the module compiles. `Module.get_attribute(__MODULE__, :attr)` returns the accumulated list at compile time.
- **`Application.start/2` callback**: Blocks supervision tree startup. Heavy initialization must happen in a GenServer's `init/1` (which runs in its own process), not in `start/2`.

## Supervision Gotchas

- **`child_spec` `id` collision**: Starting a child with the same `:id` as a running child silently terminates the old one and starts the new one. Always use unique IDs for dynamic children.
- **Max restart intensity**: `max_restarts` within `max_seconds` — if exceeded, the supervisor itself terminates. Chain goes up — if root supervisor dies, the VM exits.
- **`temporary` restart**: Never restarted, even on normal exit. Use for fire-and-forget tasks. `transient` restarts only on abnormal exit. `permanent` (default) always restarts.

## Anti-Patterns

- Blocking a GenServer's `handle_call` / `handle_cast` → offload to `Task.start` or `handle_continue`, send result back via `send`
- `Process.sleep` in tests → `assert_receive` with timeout; sleep masks race conditions and slows tests
- `try/rescue` for control flow → tagged tuples `{:ok, val}` / `{:error, reason}` are the Elixir idiom
- Global process names with atoms → memory leak (atoms are never GC'd); use `Registry` with `via_tuple`
- `Enum` on large or infinite collections → `Stream` for lazy evaluation; `Enum` materializes the entire list in memory
- Bare maps for known-shape data → structs catch missing/misspelled keys at compile time via `@enforce_keys`
- Leaking Ecto schemas across context boundaries → return plain maps or dedicated response structs from context modules
- `Repo.preload` in loops or views → N+1; push `preload` calls into the context function or use query joins
- String concatenation with `<>` in loops → accumulates binary copies per iteration; use `IO.iodata` or `Enum.map_join`
- `Code.eval_string` or `Code.eval_quoted` with user input → arbitrary code execution; never eval untrusted input
